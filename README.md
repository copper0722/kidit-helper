# KiDit 小幫手 v0.1.3

給台灣腎臟醫學會 KiDit 網站使用的開源書籤工具。v0.1 只在病人的「病史紀錄」頁，替「轉入院所」下拉選單加上搜尋框。

**護理師不必安裝 plugin、擴充功能或開啟開發者模式。** 由院所管理者在每台護理站電腦設定一次書籤，之後使用者只要登入 KiDit、開啟病人頁、點一下「KiDit 小幫手」。

## 安裝與使用

請開啟公開安裝頁：<https://copper0722.github.io/kidit-helper/>

1. 若已有舊版，管理者先從書籤列刪除「KiDit 小幫手」。
2. 在安裝頁下載 `kidit-helper-v0.1.3-bookmarks.html`。
3. Chrome 右上角選擇「⋮ → 書籤和清單 → 匯入書籤和設定」，選取下載檔；若書籤在「已匯入」資料夾，將它拖到書籤列。此方式會保留自訂水滴 favicon，不顯示預設小地球。
4. 護理師照常登入 KiDit，開啟病人的「病史紀錄」頁。
5. 點書籤列的「KiDit 小幫手」，在「轉入院所」右側輸入關鍵字後選擇結果。
6. 確認 KiDit 原本的下拉欄位，再由人工按原有的儲存按鈕。

## 安全與隱私

- 不傳送、不儲存、不記錄任何資料。
- 不載入遠端程式碼，也沒有分析或遙測。
- 不讀寫 cookie，不會自動送出表單或按儲存。
- 只在 `www.kidit-tsn.org.tw/Start/Edit/<編號>` 的病史紀錄表單執行，且只增強 `TurnHospital`；頁面結構不符時停止。
- 原始下拉選單完整保留，KiDit 仍是唯一資料處理端。

這是獨立的社群工具，**不是台灣腎臟醫學會或 KiDit 官方產品**。正式上線前，院所仍應在無真實病人資料的環境完成驗收。請勿在公開 issue 放病人資料、帳密或臨床畫面；安全問題請見 [SECURITY.md](SECURITY.md)。

## 開發

需求：Node.js 18 以上，無第三方套件。

```bash
npm test
```

`src/bookmarklet.js` 是執行碼唯一來源；建置會產生 `dist/` 測試檔、含內嵌 PNG favicon 的 Chrome 書籤匯入檔，以及 `docs/` GitHub Pages 安裝頁。授權條款為 [MIT](LICENSE)。
